Distributed Constraint Satisfaction

Ein Constraint-Satisfaction-Problem ist durch einen Algorithmus der Werte (von einen Wertenbereich) zu Variablen zuordnet gelöst. Diese Werte müssen so zugeordnet sein, dass keine Constraints oder Restriktionen verletzt sind. Mit Distributed-Constraint-Satisfaction-Problem sind die Variablen durch Agenten vertreten, die nur teile von den gesamten Problem kennen. Die Agenten kennen nur ihre Variablen und die Restriktionen die ihre Variablen betreffen. Distributed Probleme sind wichtig, wenn die Informationen nicht allen zur Verfügung gestellt werden kann. Entweder, weil es zuviel wird oder wegen Sicherheitsbedenken. Bestehende Algorithmen für Distributed-Constraint-Satisfaction können in zwei Kategorie aufgeteilt werden: backtracking und local search Algorithmen. Obwohl backtracking alle mögliche Lösungen für ein Problem finden kann, ist es oft zu langsam. Local Search ist dagegen schneller kann aber manchmal keine Lösungen für ein Problem finden obwohl eins existiert. In einen Sommerforschungsprojekt 2005 (unter der Aufsicht von Dr. Ines Arana & Dr. Hatem Ahriz), habe ich das Distributed Breakout Algorithm mit den Distributed Stochastic Algorithm verglichen. Dafür haben wir zufällig-generierten Probleme, sowie Terminplanungs Probleme genutzt. Die unverbindliche Ergebnisse haben gezeigt dass das Distributed Breakout Algorithm besser bei zufällig-generierten Probleme war und dass das Distributed Stochastic Algorithm besser bei Terminplanungs Problemen funktioniert hat. Ein Preis war für diesen Forschung zu vergeben. Zusätzlich zu meiner Forschung, hab ich Werkzeuge für weitere Experimente mit Distributed Constraint Satisfaction Algorithmen in Java entwickelt. In meiner Doktorarbeit (unter der Aufsicht von Dr. Ines Arana, Dr. Hatem Ahriz & Dr. Kit-Ying Hui), habe ich hybrid Algorithmen für Distributed Constraint Satisfaction geforscht und entwickelt. Diese Algorithmen kombinieren die Geschwindigkeit von Local Search mit der Bestätigung von Backtracking dass man immer eine Lösung findet, wenn eine existiert. Dadurch habe ich folgende neue Algorithmen für Distributed-Constraint-Satisfaction entwickelt: PenDHyb, Multi-Hyb und Multi-HDCS.

PersonalMan - Personalmanagement

PersonalMan ist eine Java Applikation die Abwesenheit Management für Arbeitnehmer und Arbeitgeber speichert. Underanderem kann der Applikation die restliche Urlaubstage, die einem Arbeitnehmer noch zusteht, berechnen und auch Ausgleichstage wegen Geschäftsreisen managen. PersonalMan werde als einfache kostenlose Lösung primär für gemeinnützige Vereine mit relativ wenig Arbeitnehmer. Die Ideen hinter PersonalMan waren in 2013 konzepiert und seitdem werde die Applikation regelmäßig für neue Java Versionen aktualisiert. PersonalMan ist ein Open Source Projekt. PersonalMan implementiert ein Client-Server-Modell. Der Server kümmert sich um die Daten und kann mit jeden Client durch ein RESTful Schnittstelle erreicht werden.

TraMS - Verkehrsmanagement Spiel

Genug gehabt von ewig in Schlangen stehen im Öffentlichen Nahverkehr, der irgendwie immer zu spät kommt? Kannst du das besser? TraMS ist deine Chance es allen zu zeigen! Es ist ein spaßiges Computerspiel wo du dein eigenes Verkehrsunternehmen führst. Du muss sicherstellen dass deine Passagiere rechtzeitig an's Ziel ankommen! Sollte dein Fahrzeug in Stau stehen, dann musst du was tun damit die Fahrzeuge wieder punktlich unterwegs sind. Sonst werden deine Passagiere frustiert sein und einen anderes Verkehrsunternehmen suchen. Schafft es dein Verkehrsunternehmen zu überleben? Oder schaffen deine Kunden dein Unternehmen zu ruinieren? Lade TraMS jetzt runter und fang an dein erfolgreiches Verkehrsunternehmen zu führen!

Eine typischer Tag in der TraMS Welt
Das Spiel geht die normalen täglichen Aufgaben eines Verkehrsunternehmen durch. Diese Aufgaben sind unter anderem:

  • Die Erstellung eines Liniennetzes und die Schaffung von genügend Fahrzeugen (Bus, Bahn usw.) um das Liniennetz effizient und pünktlich zu bedienen.
  • Tägliche Zuordnung von Fahrzeugen zu Fahrlinien, um eine Reibungslose Fahrt zu gewährleisten
  • Herausforderungen, die täglich verursachen, dass deine Fahrzeuge zu früh oder zu spät fahren.
  • Tägliche Zusammenfassung von Kundenzufriedenheit.
Am Ende jeden Tag müsst du eine bestimmte Kundenzufriedenheit erreichen sonst wirst du gefeuert!

Contacty - Contact Microservice

Microservices implementieren per definition nur ein Feature. Dieser Microservice ermöglicht die Verschickung von Emails per Java Mail durch einen Rest mit JSON Schnittstelle. Dadurch kann man durch dieser Schnittstelle und Microservice Emails von jeglicher Programmiersprache verschickt werden. Dieses Microservice unterstützt SSL. Dieser Microservice und der Quellcode kann von GitHub Repository heruntergeladen werden.

MDISCIS - MiniDISC Informationssystem

MDISCIS ist eine Java Applikation, die als Informationssystem für Minidiscs dient. Minidiscs bieten nicht die Möglichkeit Künstler und Titel als Titelinformation zu speichern. Dadurch hab ich MDISCIS in Januar 2003 als eine Lösung für dieses Problem entwickelt. Die Applikation war regelmäßig für neue Java Versionen bis Java 11 aktualisiert. Statsres - Statistikanalyse Software

Statsres ist eine Java Applikation die Statistiken wie Mittelwert, Medianwert, Standardabweichung usw. anhand von Daten berechnen kann.
